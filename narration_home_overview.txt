Welcome to the React Native PoC Documentation

Welcome to the comprehensive documentation for the React Native Proof of Concept. This application provides detailed information about a mobile application that demonstrates seamless integration of three distinct mobile capabilities: NFC tag reading, camera capture, and WebView embedding. All three features work together in a single, cohesive mobile application with bidirectional data communication.

This documentation is designed to help you understand how these technologies integrate, how the application is structured, and how each component contributes to creating a unified user experience. Whether you're a developer looking to understand the implementation details, a project manager reviewing the architecture, or a stakeholder evaluating the solution, this documentation provides the insights you need.

Key Features Overview

The React Native PoC showcases three primary capabilities, each playing a crucial role in the overall application experience.

First, we have NFC Integration. This feature enables the application to read NFC tags and display data with full NDEF support, including text and URL payloads. NFC, or Near Field Communication, allows the mobile device to interact with NFC-enabled tags and cards. In this proof of concept, users can tap their device against an NFC tag, and the application will read and display the tag's content in real-time. This capability is particularly valuable for identity verification, access control, and data transfer scenarios.

The second key feature is Camera Capture. The application can capture high-quality photos with live preview and seamless integration with the app state. When users activate the camera feature, they see a live preview of what the camera is viewing, allowing them to frame their shot perfectly. Once a photo is captured, it's immediately processed and stored within the application's state management system, making it available to other parts of the application, including the WebView component.

The third and equally important feature is Web Embedding. The application displays React web applications via WebView with real-time data synchronization from native components. This means that a web-based React application can be embedded directly within the native mobile app, and it can receive and display data captured by the native NFC and Camera components in real-time. This creates a seamless experience where web and native technologies work together as one unified application.

How These Features Work Together

What makes this proof of concept particularly interesting is how these three distinct capabilities integrate seamlessly. When a user reads an NFC tag, that data doesn't just stay in the NFC screen. Instead, it's stored in a shared state management system using React Context, and it can be immediately displayed in the WebView component. Similarly, when a photo is captured using the camera, that image information is also stored in the shared state and can be displayed in the WebView dashboard.

The communication between native components and the WebView happens through a postMessage bridge. This is a bidirectional communication channel that allows data to flow from native components to the web application, and also allows the web application to send commands back to the native components. For example, a button in the WebView dashboard can trigger an NFC scan in the native app, demonstrating true bidirectional communication.

This architecture provides several benefits. First, it allows for flexible web-based user interfaces that can be updated without rebuilding the entire native application. Second, it maintains direct access to native device capabilities like NFC and camera, which web applications cannot access directly. Third, it creates a unified user experience where users don't need to understand which parts are native and which are web-based - it all feels like one cohesive application.

Project Metadata

This React Native PoC focuses on NFC, Camera, and WebView Integration. The target platform is Android 7.0 and above, ensuring compatibility with a wide range of Android devices. The project is currently in the Documentation and Planning Phase, with comprehensive documentation and architectural planning completed.

The application is built using React Native CLI, not Expo, which provides more direct access to native modules and greater control over the build process. This choice allows for better integration with native Android features like NFC and camera, and provides more flexibility in how the application is structured and deployed.
