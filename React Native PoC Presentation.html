postMessage Bridge Architecture

Bidirectional JSON-based messaging between React Native and embedded React web app.

Native ‚Üí Web (Broadcast)

Send NFC/camera data to WebView:

webViewRef.current?.postMessage( JSON.stringify({ type: 'NFC_TAG', payload: tagText }) );

Web ‚Üí Native (Optional)

WebView requests native actions:

window.ReactNativeWebView?.postMessage( JSON.stringify({ type: 'START_NFC', payload: {} }) );

Message Schema

type: Action identifier | payload: Data object (flexible structure)

Web App: Message Listener Pattern

React component listening for native messages and updating UI in real-time.

// Inside React component in WebView import { useEffect, useState } from 'react'; export const DataDisplay = () => { const [nfcData, setNfcData] = useState(null); const [cameraData, setCameraData] = useState(null); useEffect(() => { window.addEventListener('message', (event) => { const msg = JSON.parse(event.data); if (msg.type === 'NFC_TAG') { setNfcData(msg.payload); } else if (msg.type === 'CAMERA_CAPTURE') { setCameraData(msg.payload); } }); }, []); return ( <div> <h2>Last NFC: {nfcData || 'Waiting...'}</h2> <h2>Last Camera: {cameraData?.filename || 'Waiting...'}</h2> </div> ); };

Risk Assessment & Mitigation

Risk: NFC Hardware Unavailability

Mitigation: Mock NFC responses in test mode; provide NFC tag simulator for development

Risk: Android Version Fragmentation

Mitigation: Test on Android 7.0, 10.0, 12.0, 14.0+; handle API differences in code

Risk: postMessage Latency

Mitigation: JSON serialization is fast; use debouncing if high-frequency updates needed

Risk: Camera Permissions on Older Android

Mitigation: Use react-native-permissions for uniform handling across versions
